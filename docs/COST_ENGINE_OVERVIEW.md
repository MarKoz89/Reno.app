# Cost Engine Overview

## Purpose

The cost engine produces deterministic, explainable renovation estimate ranges from structured user inputs.

AI must not set prices. AI may summarize, rephrase, or explain outputs generated by the cost engine.

## Principles

- Same inputs produce the same estimate.
- Every line item should be explainable.
- Cost ranges are better than false precision.
- Rules should be simple enough to inspect and adjust.
- Pricing assumptions should be documented before they are expanded.

## Inputs

The engine may use:

- Room type.
- Room size.
- Selected scope items.
- Current condition.
- Finish level.
- DIY comfort level.
- Professional labor requirement.
- Timeline pressure.
- Region multiplier in the future.

## Outputs

The engine should return:

- Low total.
- High total.
- Line items.
- Assumptions.
- Confidence level.
- Warnings when inputs are incomplete or unusually broad.

## Initial Cost Approach

Use rule-based calculations:

- Base cost by scope item.
- Quantity multipliers from room size.
- Condition multiplier.
- Finish/style multiplier.
- Labor multiplier when professional help is required.
- Contingency percentage.

## Example Categories

- Paint and wall prep.
- Flooring.
- Lighting.
- Fixtures.
- Cabinets or storage.
- Countertops.
- Removal and disposal.
- Contingency.

## AI Boundary

AI can:

- Explain why costs changed.
- Convert line items into plain-language summaries.
- Suggest priorities based on budget.
- Generate a renovation plan from deterministic outputs.

AI cannot:

- Invent prices.
- Override cost engine totals.
- Hide assumptions.
- Produce final estimates without structured inputs.
